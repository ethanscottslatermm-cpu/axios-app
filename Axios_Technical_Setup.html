1. Application Overview

Axios is a mobile-first personal wellness and finance platform built as a React Single Page Application (SPA). It is designed for a 430px-wide mobile viewport, centered in the browser on desktop. The app tracks health (food, water, weight, fitness), spiritual life (prayer, devotionals, scripture), finances (bills, bank accounts, stock watchlist), and a personal calendar — all under a single authenticated user account.

The app is deployed on Netlify and uses Supabase as its database and authentication backend.

State Management

Axios uses a combination of React Context (global auth, theme, app state) and TanStack Query (remote data). There is no Redux or external state library.

  • AuthContext — user session, sign-in/out, inactivity lock, heartbeat
  • AppContext — global UI state
  • ThemeContext — theme switching (Axios Platinum default + alternate themes via CSS variables)

3. Backend & Serverless Functions

There is no traditional backend server. All server-side logic runs as Netlify Functions (Node.js 18, bundled with esbuild). These functions handle anything that requires secret API credentials, keeping keys off the client.

Function File Route Purpose External Call
plaid-link-token.js /api/plaid-link-token Creates a Plaid Link token to initiate bank connection UI Plaid API
plaid-exchange.js /api/plaid-exchange Exchanges Plaid public token for access token; stores in Supabase Plaid API + Supabase (service role)
plaid-balances.js /api/plaid-balances Fetches live bank account balances for authenticated user Plaid API + Supabase (service role)
plaid-transactions.js /api/plaid-transactions Fetches last 30 days of bank transactions Plaid API + Supabase (service role)
stocks.js /api/stocks Returns live quotes for DOW, S&P 500, NASDAQ, BTC, Gold Finnhub API
news.js /api/news Aggregates headlines from CNN, BBC, NPR RSS feeds. Cached 5 min. RSS feeds (no key)
Note: All routes are mapped via Netlify's redirect rule: /api/* → /.netlify/functions/:splat. The SPA fallback (/* → /index.html) ensures React Router handles all client-side navigation.

4. Database (Supabase / PostgreSQL)

Supabase hosts a managed PostgreSQL database. Row Level Security (RLS) is enabled on every table — users can only read and write their own rows. Admin-only tables use a role check against the profiles table.

Table Purpose RLS Policy
profiles User profile: name, role, avatar, status, last_login, last_seen User owns row; admin can read all
food_logs Daily food / calorie entries with macros auth.uid() = user_id
water_logs Water intake log (oz per entry) auth.uid() = user_id
weight_logs Daily weight check-ins in lbs auth.uid() = user_id
workouts Workout sessions (date, type, duration) auth.uid() = user_id
exercises Individual exercises tied to a workout (sets, reps, weight) Nested: workout must belong to auth.uid()
prayer_logs Prayer journal entries (category, text, answered status) auth.uid() = user_id
devotionals Daily devotional reflections with scripture auth.uid() = user_id
stock_watchlist User's saved stock ticker symbols auth.uid() = user_id
calendar_events Events with type, date, time, recurring flag, email reminder auth.uid() = user_id
bills Recurring bill tracker (payee, amount, due day, autopay, paid status) auth.uid() = user_id
plaid_items Plaid bank connection tokens (access_token, item_id, institution) auth.uid() = user_id
webauthn_credentials Biometric (Face ID / Touch ID) credential IDs per user auth.uid() = user_id
login_history Login timestamps and device type per user Insert: own rows only. Select: admin only.
verse_pool Admin-managed pool of scripture references for rotating display Read: any authenticated user. Write: admin only.
app_settings Global key/value settings (e.g. maintenance mode, app version) Read: anyone. Write: admin only.

5. Authentication & Security

Supabase Auth (Primary)

  • Email + password sign-in via supabase.auth.signInWithPassword()
  • Session persisted via Supabase's built-in session management
  • Checks user status field on login — blocks inactive, deleted, and suspended accounts
  • Records last_login, last_seen, and login_history entries on each sign-in
  • Heartbeat updates last_seen every 5 minutes while the app is open

Biometric Lock (WebAuthn)

  • Face ID / Touch ID via the browser's native navigator.credentials API
  • Credential IDs stored in webauthn_credentials table
  • App automatically locks after 3 minutes of inactivity — biometric required to re-enter
  • Requires HTTPS (secure context)

Role-Based Access

  • Two roles: user (default) and admin
  • Admin role gates: admin dashboard, maintenance mode toggle, verse pool management, user management, login history
  • Finance module additionally gated behind biometric re-authentication (FinanceGuard component)

Secrets Handling

  • Client-only keys are VITE_-prefixed and safe to expose (anon key, public API keys)
  • Plaid credentials and Supabase Service Role Key are Netlify-only environment variables — never shipped to the browser
  • All tables protected by Row Level Security — no raw database access possible via the anon key beyond a user's own data
